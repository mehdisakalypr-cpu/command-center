# Wiki Farm — Specification (Hisoka Rank #3)

**Score** 59.6 · **MRR Y3** €32k · **ETA** 10-15-24j · **Worker** ultra-instinct-worker-3

## 1. Vision produit

Farm de micro-wikis multilingues SEO-first, un domaine par **niche ultra-verticale**
(ex: "wiki du café de spécialité", "wiki de la guitare manouche", "wiki du matériel
de plongée freediving"). Chaque wiki = 300-1000 articles longs (≥1800 mots) générés
par cascade IA, avec sources citées, FAQ structurée (schema.org FAQPage), et
clusters topiques autour d'un pilier d'autorité. Cible: positionnement organique
longue-traîne, monétisation ads + premium no-ads + niches premium.

## 2. Briques réutilisées

- **ai-pool cascade** (`/lib/ai-pool/`): Groq → OpenRouter → OpenAI fallback pour
  génération articles, embeddings clusters, reformulation.
- **auth-v2** (`/lib/auth-v2/`): compte premium €29/mo, WebAuthn, site_access table.
- **i18n** (`/lib/i18n/`): 10 langues cibles M1 (fr/en/es/de/it/pt/nl/pl/tr/ar).
- **Supabase** (partagé `jebuagyeapkltyjitosm`): tables `wiki_*`, RLS public read
  sur `status='live'` + `published_at IS NOT NULL`.
- **geo-pricing** (`/lib/geo-pricing/`): €29/mo modulé PPP ~195 pays.
- **Stripe subscriptions** (migration 20260423030000): plan `wiki_premium`.

## 3. Tables DB cibles

- `wiki_niches` — registre des wikis (slug, titre/desc multilingues JSONB,
  meta_keywords, tier_access free/premium, status draft/live/paused).
- `wiki_articles` — contenu (niche_id FK, slug+lang UNIQUE, title, content,
  structure JSONB avec sections/FAQ, sources JSONB array, quality_score,
  word_count, cost_eur, published_at).
- `wiki_clusters` — topic clusters (niche_id, topic, authority_score, article_ids[]).
- `wiki_scrape_cache` — cache scraping sources (source+key UNIQUE, payload JSONB,
  expires_at pour TTL).

RLS: public read sur niches `status='live'` et articles `published_at IS NOT NULL`.

## 4. Routes Next.js cibles

- `/[niche]/` — landing niche (article pilier + clusters navigables).
- `/[niche]/[slug]` — article (SSR + ISR 24h, schema.org Article+FAQPage+BreadcrumbList).
- `/[niche]/search` — recherche full-text (pg_trgm + tsvector).
- `/admin/wiki/` — CC dashboard: niches, articles, quality, coûts.
- `/admin/wiki/generate` — lancement batch (niche → N articles, worker async).
- `/api/wiki/generate` — POST cron/worker generation.
- `/api/wiki/publish/[id]` — POST publish gate (≥1800 mots + ≥5 FAQ + sources).

## 5. Gates de qualité par article

- **Longueur** ≥ 1800 mots (`word_count` calculé à l'insert).
- **FAQ** ≥ 5 questions (validation `structure.faq.length >= 5`).
- **Sources** ≥ 3 citations avec URL (validation `sources.length >= 3` + HTTP 200).
- **Dedup** Jaccard < 0.4 contre articles existants même niche.
- **Image hero** présente + HTTP 200 (réutilise publish_gate OFA).
- **Quality score** ≥ 0.7 (LLM-as-judge sur clarté/structure/véracité).

Articles qui échouent → `published_at=NULL` (draft), retry x2 puis
alerte dashboard.

## 6. Monétisation

- **Freemium**: articles accessibles tous, ads display (AdSense + direct
  sponsorship niche premium).
- **Premium €29/mo**: no-ads + accès niches premium (finance, legal, crypto)
  + téléchargement PDF + API read-only.
- **Geo-pricing**: mod PPP via `/lib/geo-pricing/` (€29 FR → $34 US → R$55 BR).
- **B2B**: licence corpus niche à revendeurs (upsell M6+).

## 7. Roadmap court

- M0 (J1-3): migration DB + routes squelette + seed 3 niches pilotes.
- M1 (J4-15): cascade generation + quality gates + 200 articles live.
- M2 (J16-24): premium gate + Stripe + 1000 articles + 3 langues.

Budget cap €4/jour pendant Phase 0 (dev), €30/jour Phase 1 (génération volume).
